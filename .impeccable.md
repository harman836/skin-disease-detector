# .impeccable.md — Design Context

## Design Context

### Users
Everyday people (patients, general public) checking a suspicious skin spot — likely on a phone, possibly anxious, with no medical training. They are not power users and are not here to admire the technology. They want a quick, credible answer and a clear next step. Mobile is the primary context. Anxiety is the default emotional state on arrival.

### Brand Personality
**Three words: Trustworthy. Empowering. Precise.**

- **Trustworthy** — The interface must signal credibility without overpromising. Language is measured, disclaimers are honest, risk levels are never dramatized.
- **Empowering** — Users leave with agency, not confusion. Every result screen answers "what should I do now?" clearly and concisely.
- **Precise** — Data is front and center. Confidence percentages, condition names, and risk breakdowns are readable and unambiguous.

The voice is calm, direct, and knowledgeable — like a trusted GP summarizing a test result, not a chatbot performing friendliness.

### Aesthetic Direction
**Clinical & trustworthy.** Clean surfaces, restrained typography, purposeful whitespace. The existing palette — navy `#2a4b7c`, teal `#489fb5`, warm cream `#faf8f5` — is correct. Fraunces (serif, headings) + Inter (sans, body) is a competent pairing.

**Dark mode:** Desirable but not blocking. Implement when practical.

**Anti-reference — must NOT look like:**
- A generic AI chatbot (dark sidebar + floating bubbles + robot avatar). The current chatbot sidebar skirts this line — it should feel like a medical consultation tool, not an LLM product demo.
- A consumer wellness app (pastel gradients, gamified elements, big emojis).

### Design Principles

1. **Clarity over cleverness.** Every interaction should be immediately understandable by someone with no tech background who is stressed. Never trade legibility for aesthetics.

2. **Information hierarchy earns trust.** The primary diagnosis dominates the screen. Secondary data (confidence breakdown, risk levels) follows in supporting roles. Users should never have to search for the answer.

3. **Actions over information.** Every result must conclude with a clear, actionable recommendation. Do not leave users with data and no direction.

4. **Restraint signals professionalism.** Fewer animations, no decorative gradients, no personality-driven micro-copy. This is a health tool — every design element must justify its presence. Silence is better than noise.

5. **Mobile-first, anxiety-aware.** Touch targets must be generous. Text must be legible in bright daylight. Loading states must be informative (not just spinners). The user holding a phone in a bathroom is the design target.

## Tech Stack (for reference)
- Backend: Flask + ONNX (YOLOv8n-cls) + Gemini 2.5 Flash
- Frontend: Vanilla JS + Jinja2 templates
- Icons: Lucide + Font Awesome 6
- Fonts: Google Fonts (Fraunces, Inter)
- CSS: Custom properties (`:root` tokens), no framework

## Existing Tokens
```css
--primary: #2a4b7c       /* Navy — buttons, links, emphasis */
--accent: #489fb5        /* Teal — secondary actions, highlights */
--background: #faf8f5    /* Warm cream */
--foreground: #2d2d2d    /* Near-black text */
--muted-foreground: #5d6d7e /* Secondary text */
--color-risk-high: to be defined (currently JS-only: #D89575)
--color-risk-medium: to be defined (currently JS-only: #D4A64A)
--color-risk-low: to be defined (currently JS-only: #4A7C7A)
```

## Known Issues (from audit)
- Risk colors live in JS only — should be CSS tokens
- `#8A8A8A` used in 3+ places instead of `--muted-foreground`
- Chat sidebar "AI Assistant" header risks generic AI chatbot aesthetic — consider reframing copy
- No dark mode yet
- Accessibility: best-effort target (WCAG AA for obvious issues, not strict compliance)
